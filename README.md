# Прогулки

Генератор пеших путеводителей по вещам, которые прячутся на виду.

Не туристический гид: фигурки в трещинах штукатурки, предметы под стеклом мостовой,
двери, на которых кто-то что-то нарисовал. У каждой точки — фотография, история,
которую можно рассказать вслух, и кнопка в навигатор.

Работает офлайн: маршрут сохраняется в телефон целиком, вместе с фотографиями.

## Запуск

```bash
pip install -r requirements.txt
python3 build.py --serve      # http://localhost:8000
```

## Добавить маршрут

```
photos/spb/moy-marshrut/01-tochka.jpg      фото
content/spb/moy-marshrut.md                описание
python3 build.py
```

Формат файла — `content/kaliningrad/_example.md`. Там же список полей.

## Добавить город

```
content/berlin/city.yml
photos/berlin/
```

## Деплой

GitHub Pages, автоматом при пуше в `main` (см. `.github/workflows/deploy.yml`).
Включить: Settings → Pages → Source: GitHub Actions.

Перед первым пушем поправьте в `site.yml` два поля:

- `repo:` — на него завязана кнопка «этого больше нет»
- `base:` — **важно.** Если репозиторий называется не `<логин>.github.io`, сайт будет
  жить в подпапке. Тогда `base: '/progulki'` (имя репозитория). Оставите пустым —
  отвалятся стили и фотографии.

## Как устроено

```
content/<город>/<маршрут>.md    текст, координаты, истории
photos/<город>/<маршрут>/       исходники (жать до 1200px)
audio/<город>/<маршрут>/        истории голосом (моно 64 kbps m4a)
theme/                          шаблоны, стили, service worker
build.py                        сборка
dist/                           то, что деплоится
```

Сборка жмёт фото в два размера (560 / 900), собирает статические страницы,
считает версию service worker'а и кладёт всё в `dist/`.

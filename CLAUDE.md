This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

# Прогулки — инструкция для Claude Code

Генератор пеших путеводителей. Контент — markdown, сборка — Python, хостинг — GitHub Pages.

## Что это за жанр

Не туристический гид. Маршруты про вещи, которые прячутся на виду: фигурки в трещинах
штукатурки, предметы под стеклом мостовой, двери с граффити, домики в дуплах.
Половина объектов — временные, их закрашивают и снимают. Это не баг, это суть.

## Тон

Живой, прямой, с мнением. Разная длина предложений. Никакого «в этом уютном месте
органично сочетается». Если объект скучный — так и сказать. Если истории нет — так
и написать: «здесь я ничего не нашёл».

**Никогда не выдумывать факты.** Всё, что попадает в текст, должно быть проверено
поиском. Если это легенда — писать «по легенде». Пользователь будет рассказывать это
вслух живым людям.

## Сборка

```bash
pip install -r requirements.txt
python3 build.py            # в dist/
python3 build.py --serve    # localhost:8000
```

Тестов и линтера нет. Проверка — сама сборка: битый `geo` или отсутствующий
фронтматтер роняют её с понятной ошибкой, отсутствующее фото печатает `! нет файла`,
но не роняет. После правки контента всегда прогонять `python3 build.py`.

## Архитектура

Весь генератор — один файл `build.py`. Пайплайн:

1. **Разбор.** Обходит `content/<город>/` (город = папка с `city.yml`). Каждый
   `*.md` (кроме `_*`) — маршрут: YAML-фронтматтер + точки. Точка — заголовок `## …`,
   парсится регексами: строки-поля (`addr:`, `geo:`, `photo:`, `audio:`, `checked:`,
   `mine:`, `fragile:`, `closed:`) и блоки `::: story|warn|look … :::`. Остальной текст —
   абзацы описания. Формат нестандартный, «настоящий» markdown-парсер не используется.
2. **Фото.** `photos/<город>/<маршрут>/<файл>` (имена — из поля `photo`) жмутся
   в два размера (560 и 900 px, JPEG q62) и кладутся в `dist/img/…` для srcset.
   Аудио из `audio/<город>/<маршрут>/` копируется как есть в `dist/audio/…`.
3. **Рендер.** Jinja2-шаблоны из `theme/templates/` (`home`, `city`, `route`,
   `quests`, база — `base.html`). Страницы — `dist/<город>/<маршрут>/index.html`.
4. **Подстановки.** `theme/app.js` и `theme/sw.js` копируются с заменой плейсхолдеров
   `__BASE__`, `__VERSION__`, `__CORE__`. Версия service worker'а — sha1 от списка
   кэшируемых страниц: изменился состав сайта → новый кэш. Правишь app.js/sw.js —
   правь исходники в `theme/`, не в `dist/` (dist сносится при каждой сборке).

Конфиг сайта — `site.yml` (`title`, `repo` — для кнопки «этого больше нет»,
`base` — подпапка на project pages). Порядок маршрутов города — поле `order`
в `city.yml`. Необязательный `quests.yml` в папке города — сквозные «квесты»
(коллекции объектов по всему городу), рендерится в отдельную страницу
`/<город>/quests/`. Пункт квеста — строка (легаси) или словарь
`name/addr/geo/note/have`: `geo` даёт точную ссылку на карту, `addr` без `geo` —
ссылку-поиск, счёт «Найдено X из Y» считается сам (поле `score` перекрывает).
Поле `how` у квеста — блок «Как проходить». Пустой кружок у пункта пользователь
отмечает сам (localStorage `progulki:qfound`), `have: yes` — красная галочка автора.
Поле `photo` у пункта — файл из `photos/<город>/quests/`, рендерится карточкой.

Деплой: пуш в `main` → GitHub Action (`.github/workflows/deploy.yml`) собирает
и выкладывает `dist/` на Pages.

## Добавить маршрут

1. Фотографии — в `photos/<город>/<маршрут>/`. Имена любые, лучше `01-название.jpg`.
2. Файл маршрута — `content/<город>/<маршрут>.md`. Формат — см. `content/kaliningrad/_example.md`.
3. `python3 build.py` → `dist/`.
4. `git push` → GitHub Action соберёт и выложит.

Новый город: папка в `content/`, внутри `city.yml`. Одноимённая папка в `photos/`.

## Координаты

Брать из Яндекс.Карт (правый клик → «Что здесь?») или через `places_search`.
**Не выдумывать и не интерполировать.** Кривая координата = человек стоит не там,
где нужно, и не находит объект.

## Что должно быть в каждой точке

- `addr`, `geo` — обязательно
- `checked` — дата, когда объект видели своими глазами
- `fragile: yes` — если может исчезнуть
- `mine: yes` — добавлено от себя, не по исходным фото (в номере будет плюс вместо цифры)
- `closed: yes` — объекта больше нет: карточка гаснет, но остаётся в истории
- Описание: что видите и почему это здесь. Без воды.
- `::: story` — байка вслух, если есть что рассказать. Крючок → поворот → короткий панчлайн.
  Последний абзац выделяется на странице, поэтому он должен быть ударным.
- `audio:` — та же story, начитанная автором (см. «Аудио — story голосом»).
- `::: look` — как искать глазами. «Смотрите на уровне пояса», «арка справа от памятника».
- `::: warn` — закрыто по понедельникам, могут снять в любой день.

Истории нужны не везде. Лучше десять сильных, чем тридцать натянутых.

## Аудио — story голосом

Аудио — не отдельный жанр, а вторая подача той же story: автор начитывает историю,
и человек либо слушает, либо разворачивает текст. Никаких длинных эссе — это стало бы
музейным аудиогидом, от которого проект отстраивается.

- Поле точки: `audio: <файл> · <длительность>`. Если у точки есть и story, и audio,
  на странице — плеер, а текст story сворачивается под «…или почитайте».
  Текст обязателен по-прежнему: многие звук не включат.
- Файлы — в `audio/<город>/<маршрут>/`. Жать до моно 64 kbps m4a:
  `ffmpeg -i in.m4a -ac 1 -b:a 64k out.m4a`. Сборка ругается на файлы больше 5 MB.
- Аудио попадает в офлайн-пакет маршрута, размер пишется на кнопке «Сохранить офлайн».
- В `sw.js` есть ручная нарезка Range-запросов для аудио — не удалять: без неё
  офлайн-звук на iOS молча не играет (Safari просит файл кусками, Cache API отдаёт целиком).

## Порядок точек

Точки в файле идут в порядке обхода. Проверяй по координатам, что маршрут не петляет:
человек не должен возвращаться туда, где уже был.

## Как устроен сайт

- Оболочка и страницы кэшируются service worker'ом.
- Кнопка «Сохранить офлайн» на маршруте кладёт все его фото в Cache API.
  После этого маршрут открывается без сети — это главное, ради чего всё затевалось.
- Отметки «нашли» — в localStorage (`progulki:found`).
- «Этого больше нет» открывает issue в репозитории с заполненным заголовком.
  Пришёл issue → поставить точке `closed: yes` или обновить.

## Пути

Все внутренние ссылки идут через `{{ site.base }}` — сайт может жить в подпапке
(GitHub Pages). Не хардкодить `/app.css` и `/img/...`, иначе на project pages всё отвалится.

## Чего не делать

- Не подключать аналитику, шрифты с CDN и прочее внешнее: сайт должен работать офлайн.
- Не класть в `photos/` исходники с телефона — жать до 1200px по ширине.
- Не менять палитру. Она одна на весь проект: штукатурка, охра, ржавчина, свинец.

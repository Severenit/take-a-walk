# Беклог

Идеи, до которых дойдут руки. Не обещания.

## Фичи

- **Фото от гуляющих** — прислать кадр к точке прямо из гида, с модерацией.
  Смысл: не галерея, а «свидетельство состояния» — подтверждение checked
  для fragile-объектов + закрытие точек без наших фото.
  - Этап 1 (0 инфраструктуры): кнопка «Нашли? Пришлите кадр» → предзаполненный
    GitHub issue, фото перетаскиванием. Модерация = разбор issues.
  - Этап 2 (когда будет поток): телеграм-бот → очередь в приватном канале →
    «принять» = автокоммит в photos/ через GitHub API.
  - Помнить: строка о разрешении на публикацию; лица и номера машин —
    проверять перед публикацией.
- **OG-теги** — чтобы ссылка красиво разворачивалась в мессенджерах.
  Нужен og:image (домен/картинка) — вернуться после появления своих фото-обложек.
- **«Обновить офлайн»** — кнопка не перекачивает маршрут, сохранённый раньше;
  добавить пере-сохранение при изменении состава.
- **Правило «рядом»** — каждая точка в тени большой достопримечательности
  упоминает её строкой в look с расстоянием. Решили не делать полем,
  а пройтись редактурой по всем точкам (не сделано).
- **Здания на схемах СПб** — слой bld для пяти питерских маршрутов
  (Overpass отдаёт центр СПб только в тихие часы; `python3 tools/fetch_map.py`
  дособерёт — но нужен force-режим или удалить json).

## Контент

- **Аудио**: сценарий «Красного кирпича» готов (`_kirpich-audio.md`) — ждёт
  начитки. Написать такие же для хомлинов и котов.
- **Хомлины**: Антошка, Уля, Марта — без своих фото и checked.
- **Петроградка (СПб)**: не пройдена — фото-заглушки чужие, заменить.
- **Западная Петроградка** — черновик `_zapad-razvedka.md`, ждёт разведки.
- **Калининград**: третий маршрут (Амалиенау/Верхнее озеро) — материала мало,
  собирается ногами. Кандидаты: `_klgd-razvedka.md`.
- **Область**: Гвардейск (люк «Загадай желание»), Черняховск (нитка улиц) —
  `_oblast-zametki.md`. Зеленоградск: незакрытые цели квестов (скамейки,
  второй светофор, знак, драконы), кандидаты `_kandidaty.md`.

- **Аудио «в дорогу» (bridge)** — поле `bridge:` у точки: второй маленький
  плеер «пока идёте к следующей». Пока решено сценарно: дорожные хвосты
  в репликах (см. _liteyny-audio.md). Механику делать, если хвосты
  приживутся на записи.
- **Поиск по сайту** — клиентский, офлайн, по нашим текстам («орёл» → две
  точки в двух городах). Дешёвая версия «спроси у гида».
- **Голосовой помощник / RAG по базе проекта** (идея Артёма, 21.07) — бот
  (телеграм, не сайт!), отвечающий ТОЛЬКО из нашей базы, с «не знаю» вместо
  выдумки. Делать после того, как базы станет заметно больше. Сайт и офлайн
  не трогать. Риск: галлюцинация от нашего имени бьёт по бренду
  «ничего не выдумано».

## Питч / бизнес

- **Миграция из open-source** (решение владельца, 21.07): свой сервер,
  закрытый репозиторий и структура. После питча. При переезде: заменить
  «этого больше нет» (сейчас GitHub issues), деплой (сейчас Actions),
  убрать «открытый проект» из футера и питча, домен.

- В питч — секция «Для кого»: формулировка Артёма про интровертов
  и самостоятельных («не гонит гид, свой темп, кафе посередине,
  аудио вместо человека»).

- Домен + редирект (QR на кассы должен вести на свой домен).
